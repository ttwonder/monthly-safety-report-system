# 月度安全會議報告系統整體計劃

## 已確認總方向

把目前的單頁 HTML demo 升級為「資料庫 + 模塊庫 + 月報編輯器 + 勾選式 PDF 輸出中心 + 歷史月報管理」。

## 第一版 V1：先做可用的模塊化月報

目標：保留現有 demo 的富文本、卡片、趨勢圖、分欄、圖標與列印風格，但增加可操作的月報工作流。

### V1 必做

1. 增加多標籤頁：
   - 月報編輯
   - 模塊庫
   - PDF 輸出
   - 歷史 / 備份
   - 第二版資料庫預留
2. 將現有 20 個固定條目視為月報模塊。
3. 每個模塊增加「是否加入 PDF」勾選狀態。
4. PDF 輸出中心只輸出已勾選模塊。
5. 支援全選、取消全選、按類別選擇。
6. 保留現有保存、Excel、data.js、富文本排版與列印能力。
7. 儲存 PDF 勾選狀態到 IndexedDB。
8. 先不重做資料庫，只為第二版預留入口。

## 第二版 V2：記錄資料庫

目標：不只是編輯月報，而是保存平時原始資料，讓月報可以從資料自動生成。

### V2 模塊

1. 檢查記錄
   - 日期、船名、檢查類型、港口、檢查方、結果、缺失數、是否滯留、附件。
2. 缺失 / 觀察項記錄
   - 關聯檢查、原文描述、中文摘要、風險等級、是否重複、改善措施、關閉狀態。
3. PSC 滯留 / 重大案例
   - 船名、港口國、港口、日期、滯留原因、原始觀察、措施、解除日期、官方 URL。
4. 行動項追蹤
   - 來源、要求、負責人、期限、狀態、證據、是否需船端回覆。
5. Safety Walk / 訓練 / 演練記錄。

### V2 已落地狀態

- 已建立 V2 HTML：`月度安全會議報告-v2.html`。
- 已加入本地 IndexedDB 記錄庫：檢查記錄、缺失 / 觀察項、PSC 滯留 / 重大案例、行動項追蹤、Safety Walk / 訓練 / 演練。
- 已加入新增、編輯、刪除、單表 CSV 匯出、整體 JSON 匯入 / 匯出。
- 已加入按月報區間統計的摘要卡。
- 已加入「用記錄資料更新月報模塊」：可更新本月檢查總覽、KPI、PSC 分析、重大異常 / 滯留、船舶回覆事項。

## 第三版 V3：KPI / 趨勢自動化

1. 自動計算本月檢查總數。
2. 自動計算各類檢查平均缺失。
3. 自動計算 PSC 滯留次數。
4. 自動產生 KPI 卡與趨勢圖。
5. 與上月 / 近 3、6、12 個月比較。

### V3 已開始落地狀態

- 已建立 V3 HTML：`月度安全會議報告-v3.html`。
- 已新增 `KPI / 趨勢 V3` 標籤頁。
- 已支援近 3 / 6 / 12 個月切換。
- 已自動生成 KPI 卡：檢查總數、缺失 / 觀察、PSC 滯留 / 重大、行動完成率。
- 已自動生成多折線 SVG 趨勢圖與趨勢表。
- 已加入「用 KPI / 趨勢更新月報」：可更新 KPI、與上月趨勢比較、Safety Walk 模塊。

## 第四版 V4：雲端共享

1. GitHub Pages 前端。
2. Supabase 資料庫。
3. Supabase RLS 權限。
4. 多人同步。
5. 操作紀錄。
6. 附件 URL / Storage 管理。

### V4 已開始落地狀態

- 已建立 V4 HTML：`月度安全會議報告-v4.html`。
- 已新增 `雲端同步 V4` 標籤頁。
- 已加入 Supabase URL / anon key / Workspace Key 設定。
- 已加入上傳本機到雲端、下載雲端覆蓋本機、完整雲端包 JSON 匯入 / 匯出。
- 已提供 Supabase RPC schema：`docs/supabase-schema-v4.sql`。
- 已提供 GitHub Pages 部署說明：`docs/github-pages-deploy-v4.md`。
- 注意：目前是簡易 Workspace Key 模式；敏感資料後續應升級 Supabase Auth + 嚴格 RLS。

## 用詞與內容要求

- PSC detention 在中文 UI 使用「滯留」。
- 保留原始觀察內容，不要只做統計摘要。
- 指標 / 統計資料要標明來源，避免 stats-first。
- 第一版先完成模塊化與 PDF 工作流；第二版再做資料庫。
